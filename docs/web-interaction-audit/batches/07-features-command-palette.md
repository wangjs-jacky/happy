## 四、根因

### 4.1 主题未接入

命令面板的根容器、输入框、结果项和空状态分别硬编码了 `white`、`black`、灰色和浅色
选中背景。组件没有从主题上下文读取语义颜色，因此系统切到深色主题后，面板仍保持
浅色外观。

### 4.2 静态命令未接入翻译

搜索框占位符已经使用翻译函数，但命令提供器中的类别、标题、副标题和空结果文案仍是
英文字符串。中文环境只能翻译输入框，无法覆盖命令数据本身。

## 五、修复方法

1. 命令面板根容器使用主题 `surface`，边框和分隔线由当前主题文字色派生。
2. 搜索框使用 `surfaceHigh`、`text`、`textSecondary`，移除硬编码浅色值。
3. 结果项的默认、选中和按压状态全部改用主题语义色。
4. 选中边框和按压背景通过颜色透明度辅助函数生成，保持主题颜色原有 alpha 语义。
5. 选中、悬停、按压和分隔线只使用会随主题包变化的语义色，避免非默认主题继承基础
   焦糖色。
6. 命令提供器复用现有导航、设置与开发工具翻译键，无标题会话也复用现有翻译。
7. 为命令类别、副标题、切换会话和空结果补充默认翻译键。
8. 所有现有语言文件补齐同构键，避免切换语言时回退或类型不一致。

## 七、使用的工具、步骤与方法

| 目的 | 工具或方法 | 本批使用方式 |
|---|---|---|
| 交互走查 | Browser Control | 复用登录标签页，对每个开关和键盘动作分别建立证据基线 |

| 安全交互 | 可逆开关 | 每个功能开关测试后立即恢复，不触发上传、删除或外部写入 |
| 隔离开发 | sibling worktree | 使用仓库同级 worktree，独立分支完成修复、验证和 PR |
| 根因定位 | 组件树 + `rg` | 从截图异常定位面板子组件，再扫描硬编码颜色和英文字符串 |
| 回归保护 | TDD | 先用隔离认证 E2E 得到精确 RED，再实现主题接入并要求全套 GREEN |
| 审查补测 | 非默认主题 + 中文语言 | Terminal 深色包检查选中态；中文浏览器检查标题、类别、副标题与空结果 |
| 提供器单测 | 翻译函数探针 | 验证静态命令与无标题最近会话都由翻译函数生成 |
| 构建兼容 | 临时 Metro 配置 | 导出时临时禁用无响应的 Watchman，成功后立即恢复并用 Git 验证零差异 |
| 交叉审查 | 独立审查 Agent | 检查跨平台样式、翻译键完整性、颜色 alpha 和 E2E 稳定性 |
