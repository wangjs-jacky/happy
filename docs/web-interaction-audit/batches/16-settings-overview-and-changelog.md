## 三、发现的问题

- `WEB-051`：公共 `Item` 组件虽然为交互行提供 button 角色，但没有默认可访问名称。
  React Native Web 会把图标字形、标题、subtitle、动态 detail 和右侧箭头共同计算为名称。
  同一入口的名称会随版本、语言、设备或状态变化，也包含读屏无意义的图标字符。

## 五、根因与修复

`Item` 已经知道每一行的 `title`、`subtitle` 与 `detail`，但 `accessibilityLabel` 只透传调用方
显式值。没有显式值时，Web 会从完整子树推导名称，因此视觉装饰也会进入交互名称。

本批修复：

1. 交互 `Item` 默认按 `title → subtitle → detail` 生成业务名称，只保留非空字符串，
   规范连续空白并去除相邻重复值。
2. 非空显式名称继续优先；显式空白名称回退到生成名称，避免制造无名控件。
3. 角色、选中状态、禁用状态与业务点击逻辑不变。
4. 更新组件测试，覆盖纯标题、组合业务名称、显式覆盖与空白回退。
5. 新增隔离认证 Web E2E，锁定设置、开发者工具清洗后的业务名称与更新日志导航。

## 七、使用的工具、步骤与方法

| 目的 | 工具或方法 | 本批使用方式 |
|---|---|---|
| 隔离开发 | `using-git-worktrees` | 创建同级 worktree；按约定先复用依赖，随后使用锁文件独立安装以保证 Metro 解析 |
| 交互走查 | `browser-control` + Chrome | 复用已有登录态，只操作设置公共入口和可逆版本交互 |
| Console 证据 | 动作时间基线 | 每个动作前记录时间，只统计其后的 warning / error |
| Network 证据 | CDP 增量 cursor | 每个动作使用独立 cursor，只统计失败数量、状态和资源类型 |
| 视觉检查 | 主内容裁剪截图 + DOM 测量 | 宽屏从主内容边界裁剪；800px 固定从 `x=250` 起，避开侧栏 |
| 外部入口 | 新标签打开后立即关闭 | 只验证发生独立打开，不读取外部页面内容 |
| 状态恢复 | 双向执行可逆交互 | 版本十连击启用一次，再十连击恢复禁用并关闭两次提示 |
| 根因定位 | `systematic-debugging` | 对照公共组件属性与 RNW 可访问树定位名称污染 |
| 回归保护 | TDD + 组件测试 + 隔离认证 E2E | 单测约束默认/覆盖规则，E2E 验证真实 Web DOM 的精确名称 |
| 启动恢复 | 临时禁用 Watchman | 本地启动时关闭 Watchman，浏览器结束后逐字节恢复配置 |
| 交叉审查 | 独立审查 Agent | 复核跨端语义、覆盖优先级、调用场景、测试和敏感数据边界 |
