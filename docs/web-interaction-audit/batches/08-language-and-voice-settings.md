## 五、根因与修复

### 5.1 使用量接口把可选数据源故障升级为页面 500

语音使用量只影响设置页展示。原 GET 路由在缺少供应商密钥或供应商请求异常时返回 500，
导致浏览器稳定记录失败请求。

修复后，协议使用明确的判别联合：

- 可用时返回 `{ available: true, ...usage }`；
- 缺少配置或上游查询失败时返回 HTTP 200 `{ available: false }`；
- 客户端只在 `available: true` 时展示用量卡片，不用伪造的零用量误导用户。

独立审查还发现，同一个上游查询函数也参与创建语音会话前的额度判断。若把供应商非 2xx
静默解释成零用量，会让会话创建检查错误放行。现在上游非 2xx 会抛出异常：GET 将其降级为
“暂不可用”，POST 则在申请会话令牌前失败关闭，继续保护免费额度、订阅和硬上限。

### 5.2 开发者区域绕过翻译系统

为 `settingsVoice` 补充静态标题、弹窗、实验选项、来源、访问方式、开关状态和三类计数
翻译键。默认英文与全部十种语言文件保持同构；实验 variant、source 和 gate 先映射为当前
语言，再拼成诊断文本。

### 5.3 移动端换行策略直接进入桌面

移动端继续用换行符显示诊断项；Web 改为中点分隔，并把状态与计数限制为最多两行。
同一组件保留跨平台语义，不增加桌面专属页面分支。

### 5.4 确认标题描述了尚未发生的结果

保留原翻译键和确认流程，只把全部语言的标题从“语言已更改”改成“需要重启”语义。
用户取消时设置保持不变。

## 七、使用的工具、步骤与方法

| 目的 | 工具或方法 | 本批使用方式 |
|---|---|---|
| 交互走查 | Browser Control | 复用登录标签页，覆盖导航、弹窗、搜索、开关、取消和状态恢复 |
| Console 证据 | 动作时间基线 | 只统计动作开始后的 warning / error |
| Network 证据 | CDP 增量 cursor | 只统计失败数量、状态和资源类型，不记录 URL、请求头或正文 |
| 视觉检查 | Browser screenshot + DOM 测量 | 截图比较语言和密度，E2E 固化状态行高度 |
| 安全交互 | 取消与可逆恢复 | 不确认跨设备语言、不选择语音语言、不重置计数、不保存代理 ID |
| 根因定位 | `rg` + 组件/路由追踪 | 从页面 useEffect 追到客户端 API 和服务端使用量路由 |
| 隔离开发 | sibling worktree | 依项目约定重新安装 pnpm workspace 依赖，不复用损坏的 node_modules |
| 回归保护 | TDD | Server 路由、翻译结构和隔离认证 Web E2E 分层覆盖 |
| 翻译审查 | i18n 子 Agent | 校验全部语言结构、术语和预确认语义 |
| 构建验证 | 临时 Node watcher | Watchman 无响应时临时切换，导出后恢复并验证无 diff |
| 交叉审查 | 独立审查 Agent | 首轮发现 1 个 Critical、2 个 Important、1 个 Minor，增量复审追加 1 个 Minor；全部关闭 |
