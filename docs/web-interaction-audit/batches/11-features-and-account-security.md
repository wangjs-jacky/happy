## 二、交互覆盖与安全边界

### 2.1 功能设置

1. 从设置页进入功能设置。
2. 记录 10 个功能开关的初始状态。
3. 逐一切换每个开关，并在同一动作后立即恢复初始状态。
4. 每次切换和恢复都分别检查 Console 与失败 XHR/Fetch。
5. 在 1681px 和 800px 视口检查标题、副标题、开关、滚动区域和水平溢出。

### 2.2 账户安全

1. 从设置页进入账户页面。
2. 可逆切换分析开关并恢复原值。
3. 打开登出确认框，检查取消和确认操作的语义，然后只点击取消。
4. 当前账户存在两个已注册推送令牌；逐一打开删除确认框后取消，不读取或记录令牌指纹。
5. 密钥的显示与复制均不执行，不读取也不输出密钥。
6. Web 上禁用的推送重新请求与重新注册操作不强制触发。
7. 当前账户没有已连接服务，因此没有可执行的断开连接确认流程。
8. 在 1681px 和 800px 视口检查隐私、推送和危险操作区域。

每次导航、切换、恢复、打开确认框和取消都分别建立 Console 时间基线与 CDP Network cursor。
只记录新增 warning / error 和失败 XHR/Fetch 的数量、状态与资源类型。

## 五、根因与修复

### 5.1 开关只有视觉标题，没有稳定名称

功能设置的 9 个未命名开关和账户分析开关现在都使用对应可见标题作为
`accessibilityLabel`。命令面板保留已有名称，因此功能页面 10 个开关均可按角色和标题唯一
定位。

### 5.2 Web 弹窗操作缺少 button 角色

`WebAlertModal` 的全部操作和 `WebPromptModal` 的取消、确认操作现在显式暴露 button 角色。
修改不改变点击处理、排列或视觉样式，只补齐原有交互所需的语义。

## 七、使用的工具、步骤与方法

| 目的 | 工具或方法 | 本批使用方式 |
|---|---|---|
| 交互走查 | Browser Control | 复用登录标签页，覆盖导航、全部开关、确认框和安全取消 |
| Console 证据 | 动作时间基线 | 每个动作前单独记录时间，只统计其后的 warning / error |
| Network 证据 | CDP 增量 cursor | 每个动作使用独立 cursor，只统计失败数量、状态和资源类型 |
| 视觉检查 | 裁剪截图 + DOM 测量 | 避开真实侧边栏，检查 1681px 与 800px 内容边界 |
| 可逆操作 | 记录原值 + 立即恢复 | 每个功能开关与分析开关都验证改变，再恢复初始值 |
| 危险操作 | 只到确认框并取消 | 登出和令牌删除不执行确认；密钥显示、复制完全跳过 |
| 隔离开发 | sibling worktree | 按全局约定创建同级 worktree；依赖符号链接失败后按锁文件本地安装 |
| 启动恢复 | 临时禁用 Watchman | Metro 初始化无响应时仅在本地关闭 Watchman，提交前恢复并验证零差异 |
| 回归保护 | TDD + 组件测试 + 隔离认证 E2E | 先固定空名称和空角色，再验证真实 DOM 与可逆恢复 |
| 交叉审查 | 独立审查 Agent | 复核语义范围、移动端兼容性、测试恢复逻辑与临时配置 |
