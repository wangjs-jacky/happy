## 五、根因与修复

React Native Web 不会根据图标、视觉按钮样式或相邻文本自动推断控件语义；原生
`accessibilityHint` 也不会自动成为 Web `dialog` 的可访问描述。工件页面的调试输出和英文
兜底文案来自早期实现，未经过正式页面的语义与本地化收口。

本批修复：

1. 浮动按钮显式要求调用方提供名称，并为工件列表行、保存、编辑和删除补齐 button 语义。
2. 抽取生成图片卡片组件，为预览与返回会话入口补齐角色、名称和禁用状态。
3. Web 弹窗使用标题作为名称；正文获得唯一 DOM ID，并通过 `aria-describedby` 与
   `dialog` 关联，同时保留原生平台的提示属性。
4. 删除工件页面的原始调试日志；新增空标题、空正文和删除失败文案并同步全部语言结构。
5. 修改不改变工件存储、图片查看、会话跳转或删除确认的业务流程。

## 七、使用的工具、步骤与方法

| 目的 | 工具或方法 | 本批使用方式 |
|---|---|---|
| 隔离开发 | `using-git-worktrees` | 按项目约定创建同级 worktree，并按锁文件安装依赖 |
| 交互走查 | `browser-control` + Chrome | 复用已有登录态，只执行空态、新建空表单、弹窗和卡片入口检查 |
| 浏览器降级 | 确定性 provider fallback | Chrome 控制通道失去响应后切换备用通道；备用通道再次超时后停止浏览器写操作 |
| Console 证据 | 动作时间基线 | 每个动作前单独记录时间，只统计其后的 warning / error |
| Network 证据 | CDP 增量 cursor | 每个动作使用独立 cursor，只统计失败数量、状态和资源类型 |
| 视觉检查 | 主内容裁剪截图 + DOM 测量 | 避开已有登录态侧边栏和卡片内容，检查宽屏与 800px 内容边界 |
| 用户数据 | 真实环境只读、隔离环境可逆 | 不写真实工件；一次性环境使用固定数据完成创建、取消删除和确认删除 |
| 根因定位 | `systematic-debugging` | 对照视觉控件、RNW DOM 与 Modal 属性映射定位语义缺口 |
| 回归保护 | TDD + 组件测试 + 隔离认证 E2E | 先获得稳定失败，再做最小角色、标签、描述关系和本地化修复 |
| 启动恢复 | 临时禁用 Watchman | Metro 初始化无响应时只在本地关闭 Watchman，提交前逐字节恢复 |
| 交叉审查 | 独立审查 Agent | 复核 RN/RNW 映射、业务等价性、翻译结构、测试副作用和敏感数据边界 |
