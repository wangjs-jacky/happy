## 三、发现的问题

- `WEB-054`：消息表格在窄屏下压缩单元格，长文件名与状态跨列重叠；代码长行被截断，页面本身也没有提供可用的横向回退。
- `WEB-055`：工具演示页使用固定浅灰背景和固定深色文字，在深色应用中形成明显的浅色页面断层。
- `WEB-056`：工具分类外层已经筛选一次，内部渲染函数又按示例 key 二次筛选，导致读取和终端只显示一个示例，其他、权限和状态显示为空。
- `WEB-057`：七组筛选没有可访问选中状态和单选组语义，并统一显示用于导航的箭头，无法表达互斥筛选的真实行为。
- `WEB-058`：只读工具卡片传入无业务效果的点击函数，导致所有卡片进入键盘焦点顺序并产生无意义日志。
- `WEB-059`：消息图片的可访问名称只放在内部图片节点，可点击的外层节点没有按钮角色和名称。

完整导航和重载另有 8 条普通 `console.log` 基线；它们不是 warning、error 或失败请求，
也不能归因于本批任一点击，因此仅作为观测记录，不扩大为跨模块日志清理。

## 五、根因与修复

本批有三类根因：富文本的固定列宽仍允许 Flex 收缩，代码文本没有保持固有宽度；
工具页的分类判断重复；演示页面与交互语义没有接入主题和可访问状态。

本批修复：

1. 表格单元格禁止 Flex 收缩，表格内容按固有宽度放入横向滚动容器。
2. 代码文本在 Web 保持预格式化和内容宽度，由代码块内部横向滚动。
3. 图片预览操作添加按钮角色和与可见替代文字一致的名称。
4. 移除工具示例渲染函数中的二次筛选，仅由分类区块决定可见示例。
5. 七组筛选组织为有名称的 `radiogroup`，每项使用 `radio` 角色、选中状态和稳定测试标识，并移除导航箭头。
6. 移除只读工具卡片的无意义点击回调。
7. 工具页使用当前主题的分组背景、主文字和次级文字令牌。

### 8.2 可避免耗时与改进

1. **依赖符号链接再次触发路由解析问题**
   - 现象：同级 worktree 复用主仓库依赖后，开发者路由回退到默认欢迎页。
   - 浪费：一次服务重启和一次依赖安装。
   - 改进：该仓库的后续 Web 走查 worktree 直接按锁文件独立安装依赖；不再重复已验证不兼容的符号链接方案。
2. **主内容操作误触已有登录态侧栏**
   - 现象：按可见文字点击时命中了侧栏预览，而不是演示主内容。
   - 处理：立即关闭并丢弃不安全画面，没有保存、转述或提交。
   - 改进：先确认主内容容器，再使用稳定测试标识或限定主内容范围；截图继续执行固定裁剪。
3. **普通日志与错误证据混淆风险**
   - 现象：完整导航稳定新增普通日志，但点击没有新增 warning、error 或失败请求。
   - 改进：账本分开记录日志级别；只把增量 warning/error 和失败 XHR/Fetch 作为当前动作失败证据。
4. **最终回归顺序**
   - 本批先完成定向 GREEN、Browser 复验和交叉审查，再运行一次最终完整 E2E，避免重复上一批的全量回归返工。

### 8.3 下批标准 SOP

1. 创建同级 worktree；本仓库 Web 走查直接按锁文件独立安装依赖。
2. 本地临时关闭 Watchman，启动 Web；提交前逐字节恢复。
3. 初始化 Browser Control 账本，按页面分组执行“基线 → 动作 → 页面/Console/Network → 安全截图”。
4. 先限定主内容范围，再点击稳定测试标识；已有登录态截图固定裁剪侧栏。
5. 测试文件放入非路由目录，先完成定向 RED/GREEN。
6. 完成 Browser 复验和独立审查后，只运行一次最终完整 E2E。
7. 串行执行完整 Vitest、Web export，避免与 CPU 密集型 E2E 并发。
8. 恢复主题、开发者模式、视口、临时配置和原页面，校验账本后再提交 PR。
