## 五、根因与修复

### 5.1 路由未进入统一标题配置

相邻设置页面都在应用布局中注册标题，使用情况路由被遗漏。现在由统一 Stack 配置使用
现有 `settings.usage` 翻译，避免页面自行维护重复标题。

### 5.2 可点击样式没有同步可访问语义

React Native Web 会让 Pressable 获得键盘焦点，但不会自动推断它是周期选择器。周期容器
现在使用 `tablist`，选项使用 `tab` 和直接 `aria-selected`；后者既会被当前 Web 版本转发，
也是当前 React Native 的跨平台属性。存在数据时的令牌/费用切换使用同一语义。

资料保存操作补充按钮角色、稳定名称和忙碌状态；姓名输入框使用可见字段标签作为名称。

### 5.3 空数据翻译存在但没有消费

组件在加载完成且数据数组为空时，使用已有的全语言 `usage.noData` 文案渲染空态，并通过
礼貌 live region 通知辅助技术。加载初值改为 `true`，避免首个请求发出前闪现假的零数据。

独立复审进一步发现认证信息和请求顺序的竞态。数据请求现在完整位于 effect 中并依赖当前
credentials、周期和会话；缺少 credentials 会结束 loading 并显示错误，effect cleanup 会
阻止已过期请求更新数据、错误或 loading 状态。

## 七、使用的工具、步骤与方法

| 目的 | 工具或方法 | 本批使用方式 |
|---|---|---|
| 交互走查 | Browser Control | 复用登录标签页，覆盖头像、文件选择器、输入恢复和周期切换 |
| Console 证据 | 动作时间基线 | 只统计动作开始后的 warning / error |
| Network 证据 | CDP 增量 cursor | 只统计失败数量、状态和资源类型，不记录 URL、请求头或正文 |
| 视觉检查 | 裁剪截图 + DOM 测量 | 避开真实侧边栏，比较标题、空态和 800px 内容边界 |
| 安全交互 | 文件选择器不上传 + 输入恢复 | 不提交资料、不上传文件、不改变跨设备账户状态 |
| 隔离开发 | sibling worktree | 发现 workspace 符号链接解析错误后，按锁文件本地安装依赖 |
| 启动恢复 | 临时禁用 Watchman | Metro 初始化无响应时本地关闭 Watchman，提交前恢复并验证零差异 |
| 回归保护 | TDD + 隔离认证 E2E | 先证明标题和 role 缺失，再验证最终 Web DOM 与可逆状态 |
| 认证路由 | 保留已有 URL 查询参数 | 避免直接拼接 pathname 时丢失隔离环境的一次性认证参数 |
| 交叉审查 | 独立审查 Agent | 复核 RN/RNW 属性转发、ARIA 组合、路由规范和测试稳定性 |
