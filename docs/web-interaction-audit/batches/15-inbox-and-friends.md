## 五、根因与修复

React Native Web 不会仅根据图标或嵌套文字生成稳定的 button 角色；placeholder 也不等于可见
标题关联。好友列表头部直接复用了“有色背景按钮”的深色文字 tint，但当前 header 本身也是
深色背景。搜索结果则把添加操作嵌套在可点击资料行中，React Native Web 会生成嵌套 button。

本批修复：

1. 收件箱、好友列表和手机收件箱返回入口显式提供 button 角色与名称。
2. 好友列表右上文字改用 header tint，不改变布局或导航逻辑。
3. 好友搜索输入框复用现有可见标题作为名称。
4. 搜索结果外层改为非交互容器；资料入口与好友操作拆成两个同级、具名 button，操作按钮继续
   公开与真实 `disabled` 同源的禁用状态。
5. 删除好友请求路径的原始 Console 成功和异常输出；用户反馈仍由现有弹窗承担。
