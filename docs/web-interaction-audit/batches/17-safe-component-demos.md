## 五、根因与修复

三个演示页直接使用了固定白色、浅灰和黑色值，没有经过主题系统；列表页在右侧插入开关时，
只设置了控件角色和状态，没有提供业务名称。

本批修复：

1. 排版页使用主题页面表面、主文字、次级文字和抬升代码块表面。
2. 颜色页使用主题页面表面、主文字和抬升代码块表面；颜色样本本身继续保留被演示的固定色值。
3. 闪烁页使用主题分组背景、页面表面、主文字和次级文字；自定义闪烁色继续保留演示值。
4. 列表开关复用现有可见标题作为 `accessibilityLabel`，不改变开关逻辑。
5. 为主题关键层级添加稳定测试标识，覆盖深色、代表性浅色和原主题恢复。

### 8.2 可避免耗时与改进

1. **交叉审查结束前启动完整 E2E**
   - 现象：首轮完整 E2E 34/34 通过后，审查新增了浅色主题和恢复断言，只能再次完整回归。
   - 浪费：约 5.7 分钟；首轮只需跑本批定向用例。
   - 改进：先完成定向 GREEN 和交叉审查，关闭 findings 后只跑一次最终完整 E2E。
2. **组合浏览器调用超时**
   - 现象：导航、等待和视口恢复放在同一次调用时多次超时并重建连接。
   - 浪费：约 2 分钟、数次重新绑定。
   - 改进：每次只执行一个导航、截图、状态读取或视口动作；优先使用轻量 DOM 控制完成重复点击。
3. **Watchman 初始化卡住**
   - 现象：首次 Web export 在 Metro 初始化约 2 分 50 秒无 CPU 活动。
   - 浪费：约 3 分钟。
   - 改进：此仓库在独立 worktree 启动浏览器或导出时，先使用本地临时 `useWatchman=false`，验证后立即逐字节还原。
4. **测试文件误入路由树**
   - 现象：页面目录中的测试文件被路由系统当成页面模块。
   - 浪费：一次浏览器复验与一次文件迁移。
   - 改进：开发者页面的组件测试统一放到 `sources/dev/`，不放在 `sources/app/`。
5. **依赖符号链接不兼容**
   - 现象：同级 worktree 复用根 `node_modules` 后路由显示空白欢迎页。
   - 浪费：一次启动诊断。
   - 改进：仍先按全局约定创建依赖符号链接；若 Metro 再现相同解析失败，再移除链接并按锁文件独立安装。

### 8.3 下批标准 SOP

1. 创建同级 worktree并按约定链接依赖；若 Metro 出现已验证的符号链接解析失败，再按锁文件独立安装。
2. 本地临时关闭 Watchman，启动 Web；提交前逐字节恢复。
3. 初始化 Browser Control 账本，按页面分组执行“基线 → 动作 → 页面/Console/Network → 安全截图”。
4. 测试文件从一开始放入非路由目录。
5. 浏览器操作拆成短调用；重复点击优先使用可见 DOM 节点。
6. 完成 RED/GREEN 后先做独立审查，再只跑一次包含审查修订的最终全量 E2E。
7. 串行执行完整 Vitest、Web export，避免与 CPU 密集型 E2E 并发。
8. 恢复所有状态和临时配置，校验账本，提交 PR 并等待 CI。
