## 三、发现的问题

- `WEB-079`：自定义二维码文本删除到空字符串时仍调用二维码编码器，产生 Console error 并使页面退回根路由。
- `WEB-080`：自定义二维码输入只有 placeholder，没有持久的可访问名称。
- `WEB-081`：17 个二维码 SVG 均没有 `img` 角色和可访问名称，读屏无法识别图形用途。
- `WEB-082`：会话编辑器的配置触发器是可聚焦 `div`，没有按钮角色、持久名称或展开状态。
- `WEB-083`：三个内联选择器没有 dialog、radiogroup、radio、选中状态和具名搜索框语义。
- `WEB-084`：会话编辑器输入框只有 placeholder，没有持久的可访问名称。
- `WEB-085`：输入文本后，发送图标启用并进入焦点顺序，但处理器为空、没有角色和名称，也没有可观察结果。
- `WEB-086`：独立审查发现 Android picker 的关闭背景会被强制设为可聚焦，却没有名称，形成无名关闭控件。

除二维码空输入动作新增 1 条 Console error 外，修复前其他安全动作新增 warning / error 均为 0/0，
失败 XHR/Fetch 均为 0。两个页面在 800px 与宽屏下均无水平溢出，会话编辑器输入区无遮挡。

## 五、根因与修复

本批有四类根因：空文本没有在二维码编码前建立边界；图形和表单控件只依赖视觉信息；
内联 picker 没有表达弹层与单选模型；尚未实现的发送动作仍被渲染为可交互控件。

本批修复：

1. 自定义文本为空时不创建二维码，显示安全空态，避免编码器收到无效输入。
2. 为二维码输入增加持久名称和稳定测试标识。
3. 为 Web SVG 与原生 Canvas 统一增加可选的二维码名称；Web 暴露 `img`，原生暴露 `image`。
4. 为 17 个演示二维码提供与页面标签对应的名称。
5. 为 Machine、Path、Agent、Model、Effort、Permission、Worktree 和折叠摘要增加按钮角色、动态名称、状态与测试标识。
6. 将 Web picker 表达为具名 dialog，选项容器表达为 radiogroup，每个选项表达为具名 radio 和实时 checked 状态。
7. 为 picker 搜索框和会话输入框增加持久名称。
8. 让 click-away 背景退出焦点顺序，同时保留鼠标关闭能力。
9. 删除空实现发送 Pressable，只保留不进入交互树的视觉演示图标。
10. 让 Android picker 关闭背景明确退出可访问树；picker 内容继续承担具名单选组语义。

### 8.2 可避免耗时与改进

1. **根目录依赖软链接不足以隔离 pnpm/Metro monorepo**
   - 现象：只复用根 `node_modules` 时 Expo 显示默认欢迎页；再软链接应用包依赖后，Metro 沿真实路径解析到 worktree 外的工作区包并构建失败。
   - 修正：停止浏览器操作和服务，只删除本批创建的两个精确软链接，执行 `pnpm install --frozen-lockfile` 独立安装并重新构建本地工作区包。
   - 下次规则：该仓库的新 worktree 直接独立安装；不要复用根或应用包 `node_modules` 软链接。
2. **静态资源 require 会让页面级 Vitest 在断言前失败**
   - 现象：会话编辑器模块加载时直接解析图片资源，Node 测试环境无法处理别名 `require`。
   - 修正：保留运行时默认资源加载，为页面组件增加可选图标源注入；测试传入无敏感内容的占位源。
   - 下次规则：含 Metro 静态资源的路由组件测试，在 RED 前先建立最小资源注入或既有资源 mock，明确区分夹具错误与产品失败。
3. **长回归只执行一次**
   - 顺序固定为定向 GREEN → Browser 修复后复验 → 独立交叉审查 → 完整 E2E/Vitest/export，避免审查修订后重复五分钟级回归。
4. **空输入必须列入文本控件的标准边界动作**
   - 自定义输入不能只测试普通修改与恢复；需要固定覆盖空字符串、逐字符清空和空态恢复。
5. **可访问树数量应检查最终浏览器语义**
   - 现象：首轮完整 E2E 为 40/42；外层具名 `role=img` 与内层 SVG 隐式图像形成 34 个节点，而 DOM 属性检查只看到了 17 个外层。
   - 修正：内层绘制 SVG 设为 `aria-hidden` 且不可聚焦，外层保留唯一具名图像；最终浏览器角色计数回到 17。
   - 改进：图形组件同时检查显式 DOM 属性和 Playwright 可访问树，防止嵌套原生元素产生重复语义。
6. **不能未经验证就覆盖加载期计数**
   - 现象：首轮测试为复用逐动作基线直接清零初始计数；独立审查指出这会掩盖晚到或新增的加载错误。
   - 修正：两个路由都精确断言隔离启动的已知脱敏证据为 0 warning、6 组 `resource-load/fetch/ERR_CONNECTION_REFUSED`，再保存动作基线并断言差值为零。
   - 改进：以后固定保存 `loadEvidence` 与 `actionEvidence`；加载数量、类别、资源类型或原因变化必须失败，不能只清空计数。
7. **首轮完整 E2E 的重复成本**
   - 浪费：一次约 5.8 分钟的 40/42，以及修订和证据复核后的重复全量运行。
   - 改进：新增语义用例先在隔离环境单独跑目标测试，并完成交叉审查，再进入全量 42 项。
8. **PR 创建需要区分本地失败与 GitHub 服务端失败**
   - 现象：远端分支和提交可读、鉴权与限额正常，但 GraphQL 与 REST 创建 PR 均返回 GitHub 服务端错误；网页回退能识别唯一表单并提交，随后也进入 GitHub 官方错误页。
   - 处理：先查询同 head 的现有 PR，确认没有重复创建；网页自动化不猜测选择器或坐标，提交失败后关闭临时标签页并保留已推送分支，继续按“查询是否已创建 → API 重试”的幂等顺序恢复交付。
   - 下次规则：外部平台出现 5xx 时，不重复提交代码、不绕过精确 head 校验；所有重试前先查询目标状态，避免服务端已成功但客户端未收到响应时生成重复 PR。

### 8.3 下批标准 SOP

1. 创建同级 worktree，并在该仓库直接独立执行冻结锁文件安装。
2. 临时关闭 Watchman，启动 Web；提交前逐字节恢复临时配置。
3. 初始化 Browser Control 账本，按“基线 → 动作 → 可观察结果 → Console → Network → 安全截图”执行。
4. 文本输入固定覆盖普通值、空值、逐字符清空和恢复。
5. picker 固定检查 trigger role/name/expanded、dialog、group、option、selected 与搜索空态。
6. 视觉上像按钮的元素必须有可观察结果；空实现从交互树和焦点顺序移除。
7. 先完成定向 TDD、Browser 复验和独立审查，再执行一次最终全量回归。
8. 恢复页面状态、视口、临时配置和原 URL，校验账本后再提交 PR。
9. 隔离 E2E 将页面加载证据与用户动作增量分开保存、分别断言。
