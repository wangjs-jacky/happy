# 桌面 Web 全量交互走查手册

> 本手册记录如何系统遍历桌面 Web 端的可交互行为，同时发现控制台异常和视觉异常。目标是让后续执行者能够重复同一套过程，而不是依赖某一次人工记忆。

## 一、适用范围

- 入口：`http://localhost:8081`
- 状态：复用浏览器已经存在的登录态，不读取 Cookie、Local Storage 或其他会话存储
- 视口：以桌面宽度为主，问题页面再补窄桌面和高分屏视口
- 交互：自动执行安全操作；修改数据、删除数据、退出登录、外部跳转等操作只记录，不自动确认

## 二、每批使用的工具

| 工具或方法 | 解决的问题 |
|---|---|
| Browser Control | 页面导航、点击、输入、弹窗检查、控制台日志采集和截图 |
| 路由静态清单 | 防止只点击当前可见入口，遗漏隐藏页面或动态路由 |
| 控制台静态扫描 | 提前定位 Web 不支持 API、弃用属性和未处理异步调用 |
| PC 视觉静态扫描 | 标记固定尺寸、绝对定位、滚动、弹层和断点等高风险区域 |
| systematic debugging | 从稳定复现和调用栈反查根因，不根据表象猜修复 |
| TDD / regression proof | 先建立失败证据，再用最小代码修复并验证回归保护 |
| 独立代码审查 | 由不同上下文检查实现、测试和移动端兼容性 |
| GitHub PR 与 CI | 每个独立修复批次单独验证、记录并合入 `main` |

## 三、安全等级

| 等级 | 定义 | 自动化策略 |
|---|---|---|
| Safe | 页面切换、展开、收起、切换只读标签、打开后可取消的弹窗 | 自动执行 |
| Reversible | 会改变本地展示设置，但能够立即恢复 | 记录原值后执行并恢复 |
| State-changing | 创建、发送、保存、上传、授权或修改远端状态 | 只走到确认前 |
| Destructive | 删除、注销、退出登录、覆盖数据 | 不执行确认动作 |
| External | 打开第三方页面、系统设置或其他应用 | 记录目标，不自动离开审计页面 |

## 四、单个交互的标准步骤

1. 记录当前 URL、页面标题、视口和必要前置状态。
2. 保存操作前截图，文件名使用 `批次-页面-操作-before.<格式>`；格式可用 PNG 或 JPEG，但同一组前后对比必须一致。
3. 记录点击前时间戳，并清点当前控制台错误和警告。
4. 执行一个交互，不把多个点击合并成一个无法定位的步骤。
5. 等待导航、动画、请求或弹层进入稳定状态。
6. 只提取点击时间戳之后新增的错误和警告。
7. 保存操作后截图，文件名使用 `批次-页面-操作-after.png`。
8. 检查 URL、焦点、滚动、遮挡、溢出、文字截断和桌面布局。
9. 将结果写入覆盖矩阵；异常分配唯一 `WEB-xxx` 编号。
10. 可逆操作恢复原状态，危险操作关闭弹窗但不确认。

## 五、异常修复步骤

1. 用同一条操作路径至少稳定复现两次。
2. 区分首个根错误和随后产生的级联日志。
3. 根据错误堆栈、事件处理器和平台分支反向追踪根因。
4. 查找同仓库内相似且正常工作的实现，比较差异。
5. 写最小失败测试；若实现已存在，用临时恢复回归证明测试确实失败。
6. 只修改根因，不顺手混入其他重构。
7. 运行定向测试、类型检查和原始浏览器路径回放。
8. 保存修复后同位置截图，视觉问题必须与异常截图使用相同视口。
9. 请求独立代码审查，解决全部 Critical 和 Important 意见。
10. 执行完整测试和 Web 构建，提交、推送并创建独立 PR。
11. 等待必需 CI；全部通过后直接合并。
12. 同步干净的 `main`，清理当前 worktree，再从新 `main` 创建下一批 worktree。

## 六、视觉检查清单

- 页面是否使用完整桌面宽度，还是被移动端最大宽度错误限制。
- 固定元素是否遮挡内容、滚动条、输入框或弹窗。
- 主内容、侧栏和右侧面板是否出现重复滚动或无法滚动。
- 弹窗、Sheet、Popover 是否超出视口或定位到触发元素之外。
- 文本是否截断但没有提示，长路径和长标题是否破坏布局。
- Hover、Focus、Disabled、Selected 状态是否清晰且一致。
- 深色和浅色主题下边界、阴影、前景文字是否可辨认。
- 窗口缩窄到断点附近时是否发生跳变、闪烁或元素重叠。

## 八、检查点

- 覆盖矩阵中的每个 Safe 交互都有执行结果。
- 每个跳过的危险交互都有理由和确认边界。
- 每个视觉问题都有异常截图和同视口修复截图。
- 每个代码修复都有失败证据、自动化回归和浏览器回放。
- 每个批次都有独立 PR、CI 结果和合并提交。
