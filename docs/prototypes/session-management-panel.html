状态优先级

  • question 事件显示为「需回答」，优先级最高。
  • permission_required 显示为「需授权」，跟在需回答后面。
  • done 且未读显示为「已完成」，用于提醒回看结果。
  • 草稿、todo、pin 不覆盖主状态，只作为辅助 badge。

排序规则

  • 排序模式只保存两个显式队列：置顶队列和需处理队列。
  • 拖进置顶队列会自动 pin；拖进需处理队列会取消 pin 并标记需处理。
  • 普通历史仍按更新时间排序，但可拖入队列或用按钮移入队列。
  • 移动端不用依赖精准拖拽，可用卡片右侧上下按钮微调顺序。
