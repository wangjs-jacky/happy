Happy Right Panel IA

Three ways to integrate Skills into a larger right-side panel

The key split is stable now: Settings remains the machine-level installed skills browser, while the right panel becomes session-level companion space. The question is how much of that companion should stay inline, how much should switch, and how much should drill into detail.

Settings

Installed skills and docs

Machine inventory, search, trigger hints, and full SKILL.md reading stay here.

Header Dropdown

Session configuration only

Agent, model, machine, folder, and permission controls stay in the existing header chip dropdown.

Right Panel

Live session companion

Skills used, touched files, generated drafts, and light actions belong here because they change with the current chat.

A

Block Stack

Everything stays in one long scroll: session summary, skills used, touched files, and quick actions all appear as consecutive blocks.

Why it fits bestThe panel stays light enough for many modules, while each module can still grow elsewhere when needed.
Clear boundaryInstalled skills remain in Settings. Used skills live here as session evidence.
Scales laterYou can add files, artifacts, run status, or actions without turning the panel into a second settings system.
Recommended split

Use Settings for machine-level browsing and documentation, keep the existing header dropdown for configuration, and turn the right panel into a session companion. In that companion, Skills should be one preview block among several, not the entire panel.
